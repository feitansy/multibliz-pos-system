## ğŸ“Š Data Directory (`data/`)

### Purpose
Centralized location for all data-related files (datasets, models, backups).

### Subdirectories

#### `data/datasets/`
- **Purpose:** Training and source datasets
- **Files:**
  - `Sample - Superstore.csv` - Original Superstore dataset (US market)
  - `Filtered_Printing_Business.csv` - Filtered for printing business (4,524 records)
- **Usage:** Used by training scripts for ML model development

#### `data/models/`
- **Purpose:** Trained machine learning models and artifacts
- **Files:**
  - `prophet_model.pkl` - Facebook Prophet time-series model
  - `xgboost_model.pkl` - XGBoost regression model
  - `feature_columns.pkl` - Feature definitions for predictions
  - `model_metadata.pkl` - Model training metadata
  - `*.png` - Training visualizations
  - `training_report.txt` - Detailed training report
- **Usage:** Loaded by forecasting app for predictions

#### `data/database/`
- **Purpose:** Database backups
- **Files:**
  - `db.sqlite3` - Database backup copy
- **Usage:** Backup and recovery

## ğŸ”§ Path Configuration

### Django Settings (`multibliz_pos/settings.py`)

```python
# Data paths configured
BASE_DIR = Path(__file__).resolve().parent.parent
DATA_DIR = BASE_DIR / 'data'
MODELS_DIR = DATA_DIR / 'models'
DATASETS_DIR = DATA_DIR / 'datasets'
DATABASE_BACKUP_DIR = DATA_DIR / 'database'
```

### Using Paths in Code

```python
# In Django views/management commands
from django.conf import settings

# Load ML model
```

## ğŸ“‹ Best Practices

### âœ… DO
- Keep datasets in `data/datasets/`
- Save trained models to `data/models/`
- Store utility scripts in `scripts/`
- Document in `docs/`
- Use relative paths from settings
- Back up database to `data/database/`

### âŒ DON'T
- Store data files in root directory
- Hard-code absolute paths
- Mix code and data files
- Store temporary files in app directories
- Commit large files to git
